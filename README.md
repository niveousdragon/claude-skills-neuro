# Скиллы для работы с Claude — для нейробиологов

Набор «скиллов» (готовых методичек для Claude Code) и короткий гайд, как с ними
работать в нейронауке: анализ данных, фигуры для статей, литература.

Техническая часть — установка, плагины, команды — вынесена в `CLAUDE.md` и
предназначена для самого Клода. Тебе её читать не нужно: открой папку в Claude
Code и попроси «поставь скиллы из этого репозитория» — он сделает всё сам.

---

## Что такое «скилл» и зачем это нужно

Скилл — это инструкция «как делать X правильно», которую Клод **сам подтягивает**,
когда видит подходящую задачу. В отличие от обычного промпта, его не надо
вставлять заново каждый раз: написал один раз — пользуется вся лаборатория,
результат одинаковый у всех.

Грубо: **промпт — сказать один раз; скилл — научить навсегда.**

---

## Как начать

1. Установи Claude Code (если ещё нет).
2. Открой эту папку в Claude Code.
3. Скажи: **«Поставь скиллы из этого репозитория».**

Клод прочитает инструкции в `CLAUDE.md`, скопирует скиллы к себе и подскажет, какие
ещё полезные наборы стоит подключить. Проверка: спроси «Какие скиллы тебе
доступны?» — должны появиться `deep-analysis` и `figure-alignment`.

---

## Что внутри — наши скиллы

### `deep-analysis` — расследование без правок кода

Изолированная сессия для разбора багов, тормозов или прототипирования. Клод
создаёт отдельную папку, гоняет эксперименты, собирает цифры и графики и пишет
резюме с выводами — **не трогая твой рабочий код**. Менять код предложит отдельно,
после твоего «да».

Когда звать:
- «Почему этот анализ считается так медленно — профилируй»
- «Сравни два метода на синтетике, дай цифры»
- «Разберись, откуда в результате эта аномалия»

### `figure-alignment` — мультипанельные фигуры для статей

Сборка фигур из нескольких панелей с **программной проверкой** выравнивания (а не
«на глаз»), квадратности и перекрытий. Лечит классическую боль: панели не на одной
линии, «квадрат» не квадратный, правка одной панели ломает другие.

Когда звать:
- «Собери фигуру 1: 5 панелей, A и B в одном ряду»
- «Панели C и D не выровнены по нижнему краю»
- «Сделай панель B физически квадратной»

## Что ещё можно подключить

Кроме наших скиллов есть проверенные сторонние наборы. Их Клод поставит по
запросу (команды — у него в `CLAUDE.md`):

- **Рабочий процесс** — как думать над задачей: разложить замысел до кода,
  дисциплина отладки, планы по шагам, «не объявлять готово без доказательств».
- **Научные статьи** — план статьи по главам, обзор литературы, **проверка цитат**
  на выдуманные ссылки, симуляция рецензии, конвертация LaTeX/DOCX/PDF.
- **Глубокий ресёрч** — основательный обзор темы с проверяемыми источниками.

Просто опиши задачу своими словами — Клод предложит нужное и поставит.

---

## Рецепты — типовые запросы

Скопируй формулировку, подставь своё.

- **Фигура для статьи:** «Собери публикационную фигуру: N панелей, такая-то
  раскладка. Используй `figure-alignment`, проверь выравнивание программно».
- **Странный результат:** «Запусти `deep-analysis`: вот что не так. Нужны цифры и
  графики, код не трогай — только предложи изменения».
- **Медленный пайплайн:** «`deep-analysis`: профилируй вот это, найди узкие места,
  сравни альтернативы».
- **Литература:** «Сделай обзор литературы по теме …», затем «помоги выстроить
  структуру раздела Methods».
- **Проверить цитаты:** «Проверь все ссылки в этом .bib/.tex на выдуманные,
  неверные годы и авторов».
- **Чужой код:** «Объясни, что делает этот модуль, нарисуй поток данных, отметь
  подозрительные места. Не меняй код».

## Принципы «как клодить»

1. **Заведи `CLAUDE.md` в своём проекте** — память проекта: как запускать тесты,
   соглашения, что нельзя трогать. Клод читает её каждую сессию.
2. **Давай контекст, а не только команду.** «Вот ошибка, вот что я ждал, вот как
   воспроизвести» сильнее, чем «почини это».
3. **Требуй доказательства.** «Готово» без прогона теста ничего не значит — проси
   вывод команды, цифры, до/после.
4. **Не давай выдумывать числа и ссылки.** Любая цифра — из реального прогона;
   любая ссылка — проверена. Нет данных — пусть скажет «не знаю».
5. **Итерируй маленькими шагами** с согласованием плана, а не «сделай всё разом».
6. **Тяжёлые расследования — через `deep-analysis`**, он не трогает рабочий код.
7. **Записывай грабли** в скиллы и `CLAUDE.md`, чтобы лаборатория не наступала на
   них снова.

## Хочу свой скилл

Скилл — это папка с файлом `SKILL.md`: шапка с `name` и `description` («когда
применять») плюс инструкция. Положи папку в `skills/` этого репозитория и
закоммить — у всех появится после обновления. Главное — грамотный `description`:
именно по нему скилл находится. За серьёзным процессом создания (с тестированием)
попроси Клода подключить набор `superpowers` и вызвать скилл `writing-skills`.

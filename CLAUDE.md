# Инструкции для Клода

Этот репозиторий раздаёт скиллы коллеге-нейробиологу. Когда пользователь
открывает эту папку и просит «поставь скиллы» (или ты видишь этот файл при старте
сессии) — выполни установку и предложи полезные плагины. Ниже — всё, что нужно.

Пользователь может быть неопытен в CLI: не заставляй его копировать команды
вручную, делай шаги сам и объясняй простыми словами.

---

## 1. Установить забандленные скиллы

В репозитории два скилла в `skills/`: `deep-analysis` и `figure-alignment`. Их
надо скопировать в личный каталог скиллов Claude Code (`~/.claude/skills/`).
Используй готовый скрипт — не копируй файлы вручную.

**Windows (PowerShell):**
```powershell
powershell -ExecutionPolicy Bypass -File .\install.ps1
```
(`-ExecutionPolicy Bypass` нужен, потому что у коллеги может быть запрещён запуск
скриптов по умолчанию.)

**macOS / Linux:**
```bash
bash ./install.sh
```

Скрипт копирует каждую папку из `skills/` в `~/.claude/skills/`, перезаписывая
старую версию, если она была. После установки скиллы доступны во всех проектах.

**Проверка:** скажи пользователю спросить «Какие скиллы тебе доступны?» — в списке
должны быть `deep-analysis` и `figure-alignment`.

> Скилл подхватывается без перезапуска: достаточно, чтобы папка появилась в
> `~/.claude/skills/`. Если не видны — проверь, что путь именно
> `~/.claude/skills/<имя>/SKILL.md`.

---

## 2. Когда вызывать забандленные скиллы

- **`deep-analysis`** — пользователь хочет расследовать баг, профилировать
  медленный код, сравнить алгоритмы, разобрать аномалию в результате. Работает в
  изолированной папке, основной код не трогает, в конце даёт handoff-резюме.
- **`figure-alignment`** — создание/правка мультипанельных matplotlib-фигур для
  публикаций: панели не выравниваются, нужен квадрат, правка одной панели ломает
  другие, общий GridSpec, программная проверка выравнивания.

Если задача подходит — вызывай скилл сам, не жди явной просьбы назвать его.

## 3. Предложить и поставить полезные плагины

Эти наборы **не лежат** в репозитории (у них свои апстримы и лицензии). Ставь их
из официальных marketplace **по запросу или когда задача попадает в триггер**.
Сначала добавляется marketplace, потом ставится плагин. Команды выполняются внутри
сессии Claude Code.

### `superpowers` — рабочий процесс (16 процессных скиллов)
```text
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers
```
Предлагай, когда пользователь: начинает новую разработку (→ `brainstorming`),
ловит баг/непонятное поведение (→ `systematic-debugging`), пишет фичу
(→ `test-driven-development`), затевает многошаговую работу (→ `writing-plans` /
`executing-plans`), или объявляет «готово» без проверки
(→ `verification-before-completion`).

### `academic-research-skills` (ARS) — статьи и литература
```text
/plugin marketplace add Imbad0202/academic-research-skills
/plugin install academic-research-skills
```
Команды `/ars-*`. Предлагай, когда речь о: структуре статьи (`/ars-plan`), обзоре
литературы (`/ars-lit-review`), проверке цитат на галлюцинации
(`/ars-citation-check`), симуляции рецензии (`/ars-reviewer`), конвертации
форматов (`/ars-format-convert`).
Лицензия CC BY-NC 4.0 — для академической работы подходит.

### `deep-research` — многоисточниковый ресёрч
Если доступен в окружении — используй для основательного обзора темы с проверкой
фактов и ссылками на источники. Отдельной установки обычно не требует; если
требует — ставится так же через marketplace соответствующего автора.

### `skill-creator` — мастер создания скиллов
```text
/plugin install skill-creator
```
Предлагай, когда пользователь хочет создать собственный скилл и нужен пошаговый
мастер. Альтернатива — скилл `writing-skills` из `superpowers` (создание по TDD).

## 4. Как создать новый скилл (если попросят)

Скилл — папка `~/.claude/skills/<имя>/` с файлом `SKILL.md`:
```markdown
---
name: имя-через-дефис
description: Use when <конкретные триггеры и симптомы, когда применять>.
---

# Название

## Overview
Что это и главный принцип в 1–2 предложениях.

## When to Use
- симптом / ситуация 1
- симптом / ситуация 2

## Workflow
Пошагово, как делать правильно.
```
Ключевое — `description`: от третьего лица, начинается с «Use when…», перечисляет
конкретные триггеры (по нему скилл и находится). Чтобы поделиться с лабораторией —
положи папку в `skills/` этого репозитория и закоммить.

Для серьёзного процесса (с тестированием скилла на субагентах) вызови
`writing-skills` из `superpowers`.

---

## Чего НЕ делать

- Не копировать сторонние плагины (superpowers, ARS) в этот репозиторий — только
  ставить из marketplace.
- Не редактировать рабочий код пользователя в рамках установки — твоя задача здесь
  только поставить скиллы и предложить плагины.

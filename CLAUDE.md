# Инструкции для Клода

> ## ⛔ АГЕНТ — ПРОЧТИ ЭТОТ ФАЙЛ ЦЕЛИКОМ, ПРЕЖДЕ ЧЕМ ЧТО-ЛИБО ДЕЛАТЬ
>
> Как только ты **склонировал, открыл или увидел** этот репозиторий — сначала
> прочитай весь этот файл, и только потом действуй.
>
> Когда пользователь просит «поставь скиллы» (в любой формулировке), это значит
> **ПОЛНУЮ установку**: кастомные скиллы из `skills/` **И** плагины из marketplace.
> Это **Шаги A + B + C** ниже — все три.
>
> **Частая ошибка:** скопировать только папки из `skills/` и остановиться. Так
> ставится меньше половины набора. Задача НЕ выполнена, пока не пройдены Шаг A,
> Шаг B и проверка (Шаг C). Не объявляй готовность раньше.

Этот репозиторий раздаёт скиллы коллеге-нейробиологу. Цель — чтобы пользователь
одним запросом получил **весь набор**, а не только два кастомных скилла.

Пользователь, скорее всего, неопытен в CLI: не заставляй его копировать команды
вручную больше необходимого, делай шаги сам и объясняй простыми словами.

---

## 1. Полная установка при первом использовании

Предупреди пользователя одной фразой, что поставишь весь набор (это подтянет
файлы из интернета и добавит команды), и, если он не против, ставь всё подряд:
сначала Шаг A, затем Шаг B, затем проверка (Шаг C).

### Шаг A. Кастомные скиллы из репозитория

В `skills/` лежат `deep-analysis` и `figure-alignment`. Их нужно скопировать в
`~/.claude/skills/` (по папке на скилл). Способ — на твой выбор:

- **Скриптом** (если можешь запускать скрипты в этом окружении):
  - Windows (PowerShell): `powershell -ExecutionPolicy Bypass -File <путь к репозиторию>\install.ps1`
  - macOS / Linux: `bash <путь к репозиторию>/install.sh`
  - (`-ExecutionPolicy Bypass` нужен, потому что у пользователя может быть запрещён
    запуск скриптов по умолчанию.)
- **Вручную** (надёжнее, если запуск чужого скрипта отклоняется): просто скопируй
  каждую папку из `skills/<имя>` в `~/.claude/skills/<имя>` своими файловыми
  инструментами. Результат тот же — это всего лишь копирование папок.

Проверка Шага A: в `~/.claude/skills/` появились папки `deep-analysis/` и
`figure-alignment/`, в каждой есть `SKILL.md`.

### Шаг B. Плагины (через marketplace)

Это остальная часть набора. Команды `/plugin` — встроенные команды сессии Claude
Code. **Ты, скорее всего, не можешь выполнить их сам** (нет такого инструмента) —
тогда их вводит пользователь.

**Как правильно отдать их пользователю (важно — здесь легко ошибиться):**

- Дай команды **по одной на строку**, и попроси вводить их **по одной за раз**:
  вставил строку → Enter → следующая.
- Предупреди: **каждая строка должна начинаться ровно с `/` (символ `/` — первый,
  без пробела перед ним)**. Пробел в начале превращает команду в обычный текст, и
  она не выполнится. Это самая частая причина «не сработало».
- Несколько команд в одну строку через пробел **не работают** — только по одной.

Команды (полный набор):

```text
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace

/plugin marketplace add Imbad0202/academic-research-skills
/plugin install academic-research-skills@academic-research-skills

/plugin install skill-creator@claude-plugins-official
```

**Почему именно так (не упрощай имена!):**

- `superpowers@superpowers-marketplace` — указывай **полное имя с marketplace**.
  Если написать просто `superpowers`, автодополнение легко соскальзывает на
  похожий `superpowers-chrome` (это браузерный инструмент, не тот плагин). Нам
  нужен именно `superpowers` — основная библиотека (TDD, debugging, планы и т.д.).
- `skill-creator@claude-plugins-official` — `skill-creator` лежит в **официальном
  встроенном** marketplace `claude-plugins-official` (он уже подключён по
  умолчанию, отдельный `marketplace add` не нужен). Без суффикса `@…` имя может не
  разрешиться.

После установки каждого плагина CLI пишет «Run /reload-plugins to apply» — попроси
пользователя в конце один раз выполнить `/reload-plugins`. Если плагины не
появятся в списке — пусть перезапустит сессию Claude Code.

> `academic-research-skills` приносит команды `/ars-*` и скилл `deep-research` —
> отдельно ставить ресёрч не нужно. Лицензия ARS — CC BY-NC 4.0, для академической
> работы подходит.

### Шаг C. Проверка (обязательно)

Сверь, что установлен **весь** набор. Лучше всего — посмотреть
`~/.claude/plugins/installed_plugins.json` и список доступных скиллов. Должны быть
все пять:

| Компонент | Где живёт | Как проверить |
|---|---|---|
| `deep-analysis` | `~/.claude/skills/deep-analysis/SKILL.md` | папка на месте |
| `figure-alignment` | `~/.claude/skills/figure-alignment/SKILL.md` | папка на месте |
| `superpowers` | плагин `superpowers@superpowers-marketplace` | есть в `installed_plugins.json` |
| `academic-research-skills` | плагин `…@academic-research-skills` | есть в `installed_plugins.json` |
| `skill-creator` | плагин `skill-creator@claude-plugins-official` | есть в `installed_plugins.json` |

Если чего-то не хватает — вернись к нужному шагу и доставь. Только когда все пять
на месте, сообщи пользователю, что установка завершена, и предложи проверить
самому: пусть спросит «Какие скиллы тебе доступны?».

> Скиллы из `skills/` подхватываются без перезапуска — достаточно, чтобы папка
> появилась в `~/.claude/skills/<имя>/SKILL.md`. Плагинам может понадобиться
> `/reload-plugins` или перезапуск сессии — предупреди об этом.

## 2. Что когда вызывать (после установки)

### Кастомные скиллы
- **`deep-analysis`** — расследовать баг, профилировать медленный код, сравнить
  алгоритмы, разобрать аномалию в результате. Работает в изолированной папке,
  основной код не трогает, в конце даёт handoff-резюме.
- **`figure-alignment`** — создание/правка мультипанельных matplotlib-фигур для
  публикаций: панели не выравниваются, нужен квадрат, правка одной панели ломает
  другие, общий GridSpec, программная проверка выравнивания.

### Из `superpowers`
- `brainstorming` — перед новой разработкой, разложить замысел до кода.
- `systematic-debugging` — любой баг/непонятное поведение.
- `test-driven-development` — пишется фича.
- `writing-plans` / `executing-plans` — многошаговая работа с чекпойнтами.
- `verification-before-completion` — пользователь объявляет «готово» без проверки.

### Из `academic-research-skills`
- `/ars-plan` — структура статьи по главам.
- `/ars-lit-review` — обзор литературы.
- `/ars-citation-check` — проверка цитат на галлюцинации.
- `/ars-reviewer` — симуляция рецензии.
- `/ars-format-convert` — конвертация LaTeX / DOCX / PDF / Markdown.
- `deep-research` — основательный обзор темы с проверяемыми источниками.

Если задача подходит — вызывай нужное сам, не жди, что пользователь назовёт скилл.

## 3. Как создать новый скилл (если попросят)

Пользователь, скорее всего, не пользуется терминалом и git — **бери всю механику
на себя**. Не проси его создавать файлы или коммитить вручную. Алгоритм:

1. Выясни у пользователя своими словами две вещи: **когда** приём нужен (триггеры,
   ситуации, формулировки) и **как** делать правильно (шаги). Задавай вопросы,
   если чего-то не хватает.
2. Сам создай файл `~/.claude/skills/<имя>/SKILL.md` по шаблону ниже.
3. Проверь, что скилл подхватился, и кратко объясни пользователю, как его звать.
4. Если пользователь хочет поделиться с лабораторией — скопируй папку скилла в
   `skills/` этого репозитория и сделай коммит сам (`git add` + `git commit`).
   Сообщи, что остальным он станет доступен после обновления репозитория.

Шаблон `SKILL.md`:
```markdown
---
name: имя-через-дефис
description: Use when <конкретные триггеры и симптомы, когда применять>.
---

# Название

## Overview
Что это и главный принцип в 1–2 предложениях.

## When to Use
- симптом / ситуация 1
- симптом / ситуация 2

## Workflow
Пошагово, как делать правильно.
```
Ключевое — `description`: от третьего лица, начинается с «Use when…», перечисляет
конкретные триггеры (по нему скилл и находится).

Для серьёзного процесса (с тестированием скилла на субагентах) вызови
`writing-skills` из `superpowers`, либо мастер `skill-creator`.

---

## Чего НЕ делать

- Не копировать сторонние плагины (superpowers, ARS, skill-creator) в этот
  репозиторий — только ставить из marketplace.
- Не ставить `superpowers-chrome` вместо `superpowers` — это разные плагины (см.
  Шаг B). Если поставил по ошибке, удали: `/plugin uninstall superpowers-chrome@superpowers-marketplace`.
- Не редактировать рабочий код пользователя в рамках установки — твоя задача здесь
  только поставить набор скиллов и плагинов.
- Не останавливаться на Шаге A. Установка без плагинов (Шаг B) — это половина
  набора, а не готовый результат.
